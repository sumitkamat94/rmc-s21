Broad area/overview

in this paper it was shown that manipulator is not an isolated systems and it is in interaction with its environment. The approach presented in this book is taking the mechanics of interaction between the manipulator and physical system into account. also, it was shown that for robot manipulator, position and force control are not sufficient and control of the dynamic behavior of the system is also required. It is shown that in order to model this interaction we need nonlinear equations, based on this fact that the environment and the manipulator should complement each other and because the environment contains inertial objects, which makes the environment admittance, the manipulator must be impedance. For a broad class of nonlinear systems a general network is presented which makes it possible to control motion and impedance separately.

Specific Problem

in this paper impedance control is defined as a unified approach for manipulation.it was shown the way that the controller may modify the behavior of manipulator by using the network concepts of Bond graphs.

Basic Ideas and definitions

  • The manipulator is a set of physical structures, sensors, and actuators (hardware) combined with some set of control algorithms (software). A unified approach to take both the hardware and the software behavior into account can be acquired by using the assumption that no controller can make the manipulator appear to the environment as anything other than a physical system.

  • in this paper admittances is defined as a system accepting effort (e.g., force) as inputs and yielding flow (e.g., motion) as outputs; and impedances accepting flow (e.g., motion) as inputs and yielding effort (e.g., force) outputs.

  • The distinction between admittance and impedance is fundamental: and it is not an interchangeable.

  • these physical systems should be complementary to each other. it means that along any degrees of freedom, if one is an impedance, the other must be an admittance and vice versa.

  • The impedance might be modulating during different parts of the tasks. therefore, the controller should be able to deal with this issue.

  • The simplest equivalent physical network representing impedance control is shown in the following figure. The position commanded by the high-level supervisor is represented by a flow source. To prevent causal conflict between this element and the environmental admittance (which must experience an impressed effort) a zero junction is interposed between the two. The impedance coupled to this zero-junction represents the relation between force and motion commanded by the supervisor and includes both the static force/displacement relation plus the possible dynamic terms required to ensure controlled dynamic behavior.

Conclusion

  • The most interesting consequence of the assumptions underlying impedance control is that if the dynamic behavior of manipulator is decoupled into set of components, all these separated components can be added by a simple addition. This is the result of assuming that the environment is an admittance.

  • When the manipulator is decoupled from its environment the terms in the dynamic equations due to the environmental admittance disappear and in principle the manipulator alone need exhibit no inertial behavior. In practice the uncoupled manipulator still has inertia. Because of the inevitable inertial dynamics of the isolated manipulator the superposition of impedances holds even when the manipulator is uncoupled from its environment as there is always an admittance to sum forces and impedances.

Comments

  • in the impedance control, the dynamic equations of the manipulator are used which make it multi-input multi-output systems which are more complicated.

  • this approach also adds nonlinearity to the equations which is again more difficult to deal with.
